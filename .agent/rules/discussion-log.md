# 讨论记录规范

## 生效路径

| 路径 | 原因 |
|------|------|
| `docs/**` | 讨论记录文件存放于 `docs/<module>/discussion.md` |
| `**/*`（长时间讨论时） | 任何模块的深度讨论均可触发，不受路径限制 |

## 目的

在长时间架构/方案讨论中，定期精炼对话要点，让下一个接手 agent 无需重读全量聊天记录即可快速理解上下文。

## 触发条件

以下情况必须创建或更新讨论记录：

1. 讨论超过 **20 轮** 尚未结束
2. 做出了 **3 个以上** 关键决策
3. 用户说 "先记下来" / "备忘" / "后面继续讨论"
4. 当前 session 结束前有未完成的方案讨论

## 文件位置

```
docs/<module>/discussion.md
```

每个模块一个文件，追加式记录（不覆盖历史）。

## 格式

```markdown
# <模块> 讨论记录

## [YYYY-MM-DD HH:mm] 第 N 轮：<简短主题>

### 参与方
- 用户、Agent

### 已决策

| # | 决策 | 理由（一句话） |
|---|------|---------------|
| 1 | 树结构只留 parentId | children 数组从未被查询使用，双向维护易不一致 |
| 2 | status 双维度拆分 | 认知流程和发布控制是独立维度 |
| ... | ... | ... |

### 待决策

- [ ] 编辑器方案（Tiptap 换 Milkdown vs 保留）
- [ ] ...

### 关键上下文（给下一个 agent）

- 当前分支：`feat/note-model-refactor`
- 相关 PRD：`docs/note/PRD.md`（已更新）
- 核心矛盾：编辑器输出 HTML vs 存储 Markdown
- 已有代码：Tiptap 编辑器 + 乐观更新缓存层，见 `client/src/features/note/model/cache.ts`

### 讨论摘要

3-5 句话简述本次讨论的核心脉络。
```

## 使用方式

1. Agent 讨论过程中，每当累积 3 个以上决策时，主动更新 `discussion.md`
2. 下一个 Agent 接手时，先读 `docs/<module>/discussion.md` 再读 PRD
3. 决策落地后（已编码），在对应条目后标注 `✅ done`
4. 讨论记录不要求单独 commit；一旦随代码或文档提交，仍必须记录到 `docs/changes/YYYY-MM-DD.md`

## 禁止

- 不要把大段聊天原文贴进去——只记决策和上下文
- 不要创建多个分散的讨论文件——每个模块一个
- 不要用 discussion.md 替代 PRD——讨论记录是过程，PRD 是结论
