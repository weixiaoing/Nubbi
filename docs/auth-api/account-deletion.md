# 注销账号流程

## 功能目的

注销账号是不可恢复的危险操作，必须避免误触直接删除账号。因此前端需要提供二次确认，并通过当前账号邮箱验证码完成最终验证。

## 用户可见行为

1. 用户在头像气泡列表中点击 `注销账号`。
2. 系统先弹出二次确认框，提示注销会删除账号、登录会话以及个人数据。
3. 用户点击 `继续验证` 后，进入邮箱验证码弹窗。
4. 用户点击 `获取验证码`，验证码发送到当前账号邮箱。
5. 用户输入 6 位数字验证码，并勾选“我确认要注销当前账号，并了解该操作不可恢复”。
6. `确认注销` 按钮才允许提交，提交成功后账号被删除并返回登录状态。

## 关键技术设计

- 前端头像菜单入口统一调用二次确认弹窗，确认后再打开 `AccountDeletionModal`。
- `AccountDeletionModal` 负责验证码发送、验证码输入、不可恢复确认勾选和最终注销提交。
- 后端通过 `account_deletion_codes` 存储注销验证码，验证码 10 分钟有效，60 秒内不能重复发送。
- 最终注销接口校验当前登录用户、不可恢复确认标记、邮箱验证码和验证码有效期后，删除用户相关会话、账号、笔记、文件记录、文件夹、上传任务和会议评论。

## API

### `POST /auth/account/delete/send-code`

- 鉴权：需要当前登录态或 Bearer token。
- 请求体：`{}`
- 成功：返回当前账号邮箱、60 秒发送冷却和 10 分钟有效期。
- 失败：
  - `401`：未登录。
  - `400`：当前账号没有可用邮箱。
  - `429`：60 秒内重复获取验证码。
  - `500`：邮件发送失败。

### `POST /auth/account/delete/confirm`

- 鉴权：需要当前登录态或 Bearer token。
- 请求体：`{ code: string, confirmed: true }`
- 成功：删除账号和账号数据，前端清理登录态并返回登录页。
- 失败：
  - `401`：未登录。
  - `400`：未传 `confirmed: true`、验证码不是 6 位数字、验证码错误或已过期。

## 注意事项

- 不能绕过二次确认直接打开最终删除操作。
- 不能在未输入有效验证码或未勾选不可恢复确认时允许提交；后端也必须拒绝未传 `confirmed: true` 的确认请求。
- 注销验证码只用于当前登录账号，不复用注册、邮箱验证或忘记密码验证码。
- 物理文件只有在没有其他文件记录引用同一路径时才删除。
