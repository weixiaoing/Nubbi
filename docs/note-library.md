# Notes 库页

## 功能目的

Notes 库页用于集中查看和管理当前登录用户的全部 note。它和侧边栏树不同：侧边栏仍按根节点展开层级，库页则默认展示所有根节点、父节点和子节点，方便按更新时间扫描、搜索、批量移动或删除。

## 用户可见行为

- 页面标题显示为 `Notes`。
- 默认列表展示全部 note，而不是只展示根级 note。
- 列表支持按名称和更新时间排序，搜索框只过滤当前全量列表。
- 普通点击列表行会切换选中状态；点击标题进入就地重命名，点击右侧“打开”按钮才进入 note 详情。
- 多选后，批量操作栏显示在列表表头内。
- 点击“导入 Markdown”可以一次选择多个 `.md` / `.markdown` 文件，系统会批量创建根级 note。
- 点击“移动”会打开类 Notion 的轻量浮层：
  - 顶部输入框用于搜索移动目标。
  - 未输入关键词时显示最近编辑的 10 条 note。
  - 有子 note 的候选项可以像侧边栏一样展开，继续查看和选择子级目标。
  - 输入关键词后显示搜索结果。
  - 不显示多库切换入口，因为当前产品只有一个 Notes 空间。
- 添加子文档弹窗里的 `Add to` 使用同一套目标选择器，默认推荐最近 note，搜索时可切换到任意匹配的 note。

## 关键技术设计

- 库页通过 `GET /note/all` 获取当前用户全部 note，接口需要登录，返回不包含 `content` 的摘要列表。
- 侧边栏仍使用 `GET /note/roots` 和 `GET /note/children`，不受库页全量列表影响。
- 移动弹窗的默认推荐优先使用 `GET /note/recent` 数据；如果最近列表为空，则回退到库页全量列表并按更新时间排序。
- 搜索移动目标继续使用 `POST /note/search`，以便搜索到不在最近列表中的目标。
- Markdown 导入在前端完成文件读取和解析：
  - 标题优先取 frontmatter 中的 `title`，其次取第一个一级标题，最后使用文件名。
  - frontmatter 会从正文中移除，并写入 note 的 `meta`；导入内容仍保持 Markdown 格式，由 Tiptap Markdown 扩展渲染。
  - 批量导入会复用 `createNoteAtom` 逐个创建根级 note，并在完成后刷新全量列表。
- 目标选择 UI 抽为 `NoteTargetPickerPanel` 和 `NoteTargetPickerOverlay`：
  - `NoteTargetPickerPanel` 负责搜索框、最近列表、搜索结果、选中态和空状态。
  - `NoteTargetPickerOverlay` 只负责页面级移动弹窗的遮罩和居中定位。
  - 添加子文档的 `Add to` 直接在 popover 内复用 `NoteTargetPickerPanel`。
- note 列表缓存新增全量列表 key；新建、删除、移动和属性更新后会失效全量列表、根列表、子列表和最近列表，避免刷新后数据不一致。

## 移动限制

- 前端会基于全量 note 的 `parentId` 链过滤非法目标：
  - 当前正在移动的 note 自身不可选。
  - 当前正在移动的 note 的子级、孙级等后代不可选。
- 后端在 `PUT /note/properties` 中再次调用 `validateNoteMoveTarget()` 校验目标父链，防止绕过前端造成循环结构。
- 如果后端拒绝移动，前端会显示接口返回的错误信息，例如“不能移动到自身或子级”。

## 注意事项

- `GET /note/all` 是库页专用的登录态接口，不替代历史公开接口 `GET /note/getNote`。
- 全量列表不返回正文内容，打开单篇 note 时仍通过详情接口加载正文。
- Markdown 导入后的正文在 Tiptap 编辑器中展示；编辑器关闭浏览器拼写检查，避免英文测试内容出现系统自动加的波浪线。
- 当用户选中了所有可见 note 时，移动弹窗可能显示“暂无可移动的位置”，这是因为所有候选目标都被过滤掉了。
