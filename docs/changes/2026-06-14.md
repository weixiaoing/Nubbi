## [晚] commit: `待提交` - 功能(auth)：双重 token 刷新机制，减少频繁重新登录

**提交者**: weixiaoing
**变更文件**:
- server/app/lib/auth.ts（修改）
- server/app/index.ts（修改）
- server/app/middleware/session.ts（修改）
- client/src/utils/auth.ts（修改）
- client/src/api/request.ts（修改）

**Review 结果**: ✅ 通过

### 变更摘要
- **Session 延长**：`expiresIn` 从 7 天延长至 30 天，`updateAge` 从 1 天调整为 7 天
- **JWT 插件（服务端）**：添加 `jwt()` 插件，每次 `getSession()` 响应通过 `set-auth-jwt` 头下发 15 分钟短生命 JWT（EdDSA）；CORS 同步暴露该头
- **JWT 中间件验证（服务端）**：`requireAuth` 优先用 `jose.jwtVerify()` 无状态验证 JWT（JWKS 缓存 1 小时，无 DB 查询），bearer 不是 JWT 格式才降级到 `auth.api.getSession()` 查 DB
- **双 token 客户端（client）**：`runtimeState` 新增 `jwtToken` / `jwtExpiresAt`；`onSuccess` 从 `set-auth-jwt` 提取并存储；新增 `ensureJwt()` 函数——有 session 无 JWT 或 JWT 剩余 < 2 分钟时自动调 `getSession()` 刷新，随后所有 API 请求的 bearer 改用 JWT，session token 只用于 authClient 内部鉴权调用
